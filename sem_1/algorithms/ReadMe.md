# Вариант 1

## Оптимизация маршрутов

Логистическая компания столкнулась с проблемой планирования маршрутов для своего автопарка.
Маршрут представляет собой путь между городами, который должен удовлетворить нескольким критериям одновременно:

1) быть коротким по протяжённости,
2) быстрым по времени,
3) дешёвым по стоимости.

Программа получает входные данные из файла “input.txt”. После этого вычисляет кратчайший маршрут по одному из параметров, используя алгоритм Дейкстры. 
Проводится выбор одного из найденных маршрутов, согласно приоритетам параметров, заданных в строке запроса - это и есть компромиссный маршрут.
После этого, программа формирует в файл "output.txt" следующие данные:

- Три оптимальных маршрута по отдельности по каждому критерию.
- Итоговый компромиссный маршрут, выбранный на основе приоритетов.

Пример входных данных:
```
[CITIES]
1: Москва
2: Санкт-Петербург
3: Нижний Новгород
4: Казань
[ROADS]
1 - 2: 700, 480, 800
1 - 3: 400, 250, 300
2 - 3: 1100, 700, 1200
3 - 4: 350, 300, 500
1 - 4: 800, 600, 1000
[REQUESTS]
Москва -> Санкт-Петербург | (Д,В,С)
```

Пример выходных данных:
```
Запрос 1
ДЛИНА: Москва -> Санкт-Петербург | Д=700, В=480, С=800
ВРЕМЯ: Москва -> Санкт-Петербург | Д=700, В=480, С=800
СТОИМОСТЬ: Москва -> Санкт-Петербург | Д=700, В=480, С=800
КОМПРОМИСС: Москва -> Санкт-Петербург | Д=700, В=480, С=800
```
