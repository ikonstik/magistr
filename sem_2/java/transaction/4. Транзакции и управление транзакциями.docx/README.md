# Банковское приложение для управления счетами

Java-приложение для работы с банковскими счетами и переводами между ними с использованием транзакций в базе данных.

## Описание

Приложение демонстрирует использование JDBC и транзакций для управления банковскими счетами. Включает в себя:

- Создание банковских счетов
- Переводы денег между счетами с использованием транзакций
- Просмотр информации о счетах
- Валидацию данных
- Консольный интерфейс

## Технологии

- Java 11
- H2 Database (in-memory)
- JDBC
- Maven

## Структура проекта

```
src/main/java/com/bank/
├── BankApplication.java          # Главный класс приложения
├── model/
│   └── Account.java             # Модель банковского счета
├── database/
│   └── DatabaseManager.java     # Управление подключением к БД
├── service/
│   └── AccountService.java      # Сервис для работы со счетами
└── ui/
    └── BankConsole.java         # Консольный интерфейс
```

## Требования

- Java 11 или выше
- Maven 3.6 или выше

## Установка и запуск

1. Клонируйте репозиторий или скачайте файлы проекта
2. Откройте терминал в папке проекта
3. Скомпилируйте проект:
   ```bash
   mvn compile
   ```
4. Запустите приложение:
   ```bash
   mvn exec:java -Dexec.mainClass="com.bank.BankApplication"
   ```

## Использование

После запуска приложения вы увидите консольное меню с опциями:

1. **Создать новый счет** - создание нового банковского счета
2. **Перевести деньги между счетами** - перевод средств с одного счета на другой
3. **Просмотреть информацию о счете** - получение информации о конкретном счете
4. **Просмотреть все счета** - отображение всех существующих счетов
5. **Выход** - завершение работы приложения

## Особенности реализации

### Транзакции
- Метод `transferMoney` использует транзакции для обеспечения атомарности операций
- При ошибке транзакция автоматически откатывается
- Проверка достаточности средств перед выполнением перевода

### Валидация данных
- Баланс счета не может быть отрицательным
- Сумма перевода должна быть больше нуля
- Имя владельца счета не может быть пустым
- Нельзя переводить деньги на тот же счет

### База данных
- Используется H2 in-memory база данных
- Автоматическое создание таблицы `accounts` при запуске
- Singleton паттерн для управления подключением к БД

## Пример использования

1. Создайте первый счет:
   - Имя владельца: "Иван Иванов"
   - Начальный баланс: 1000.0

2. Создайте второй счет:
   - Имя владельца: "Петр Петров"
   - Начальный баланс: 500.0

3. Переведите 200.0 с первого счета на второй

4. Просмотрите информацию о счетах для проверки результата

## Обработка ошибок

Приложение корректно обрабатывает следующие ошибки:
- Недостаточно средств на счете
- Счет не найден
- Некорректные входные данные
- Ошибки подключения к базе данных
- Ошибки выполнения SQL-запросов
