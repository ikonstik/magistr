# Booking-сервис

## calc_price
### Рассчитывает итоговую сумму за count билетов с учетом базовой цены и возможной скидки.
```
Параметризованный тест с проверкой:
- соответствие типов входных данных
- соответствие значений и граничных случаев
- соответствие значений с учетом скидки или без нее зя count билетов
12 тестов
```

## check_availability
### Проверяет наличие свободных мест. При нехватке мест возвращает False, иначе — True
```
Параметризованный тест с использованием mock-объекта
- соответствие типов входных данных
- соответствие значений для event_id, seats_requested - не отрицательны
- проверка возможности забронировать больше мест чем доступно
- обработка ошибки подключения к БД
8 тестов
```

## apply_promo_code
### Применяет промокод к заказу, если он действителен.
```
Параметризованный тест с использованием фикстуры для PromoRepo
и использованием mock-объекта
- соответствие типов входных данных
- соответствие значений (order_id > 0)
- проверка на лимит использования промо
- проверка на корректность введенного промо
- проверка на использование неактивного промо
- обработка ошибки подключения к БД
8 тестов
```

## generate_booking_ref
### Генерирует уникальный код/референс бронирования на основе идентификаторов пользователя и события.
```
Параметризованный тест с использованием фикстуры
- соответствие типов входных данных
- соответствие значений (user_id, event_id > 0)
- проверка на оплату мероприятия
- проверка на уникальность результата при повторных вызовах
11 тестов
```

## send_notification_email
### Отправляет уведомление на почту при успешном бронировании/оплате
```
- валидация email
- проверка оплаты
- обработка ошибки соединения с сервером
4 теста
```
